# Important Discovery About Dataset

High-score comments were often:

* Jokes
* Memes
* Community humor

Example:

```text
"Now you're in a Regret Room"
```

Without context this is meaningless.

Conclusion:

```text
Score != Importance
```

Do not use score alone.

\---

# Context Problem

Many comments only make sense with the title.

Bad:

```text
Comment only
```

Good:

```text
Title + Comment
```

Example:

```text
Title:
My first mechanical keyboard ruined all others

Comment:
Now you're in a Regret Room
```

Now AI understands the context.

# Final Analysis Strategy

Originally considered:

### Approach 1

```text
Title + Comment
↓
Classify
↓
Rank
```

Problem:

One discussion with many comments can dominate results.

\---

### Approach 2

```text
Discussion ID
↓
All comments
↓
Analyze discussion
↓
Aggregate
```

Preferred direction.

# Final Recommended Architecture

## Step 1

Group dataset by:

```text
ID
```

Each ID represents one discussion.

Structure:

```text
Discussion
├── Title
├── Comments
├── Score
└── URL
```

\---

## Step 2

For each discussion:

Send:

```text
Title
+
All Comments
```

to Ollama.

Prompt:

```text
Analyze this discussion.

Return:

- Pain Points
- Gains
- Personas
- Must-Haves
- Nice-To-Haves
- Feature Requests

Rate importance from 1-10.

Return JSON only.
```

\---

## Step 3

Store Results

Example:

```json
{
  "discussion\_id": "123",
  "pain\_points": \[
    {
      "theme": "Bluetooth Reliability",
      "importance": 9
    }
  ]
}
```

\---

## Step 4

Aggregate Across Discussions

Instead of counting comments:

Use:

```text
Number of discussions mentioning a theme
```

Example:

Bad:

```text
Bluetooth mentioned 500 times
```

Good:

```text
Bluetooth discussed in 42 discussions
```

This avoids one viral thread dominating results.

\---

## Step 5

Create Global Theme Rankings

Merge similar topics:

Example:

```text
Bluetooth
Wireless Issues
Pairing Problems
Disconnects
```

becomes:

```text
Bluetooth Reliability
```
